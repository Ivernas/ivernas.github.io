sudo

发表于 2017-07-27 | 分类于 linux | 阅读次数

之前用的系统sudo一直是好用的，所以以为sudo是系统内置的命令。今天重装了一个比较干净的debian，发现使用sudo时提示“command not found”，才知道sudo也是一个后安装的工具。
所以，解决这个问题需要：
1.apt-get install sudo
2.安装好之后，用root权限修改文件／etc／suoders。
模仿root的权限给予自己的账户sudo的权限，如gai ALL=(ALL:ALL)ALL

格式说明[用户名 网络中的主机=（执行命令的目标用户:执行命令的用户组）执行的命令范围
如果想了解更详细的权限控制，参看man sudoers

sudo的幕后过程

用sudo执行命令时，是在当前进程的基础上fork出一个子进程，这个子进程是以root权限执行的。
然后在子进程中，执行你在sudo后面跟的命令。

执行sudo cd等命令时显示“command not found”的原因

sudo后fork出了子进程，子进程无法获取父进程所在的目录，所以无法执行cd命令。
