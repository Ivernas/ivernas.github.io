go学习笔记1

发表于 2017-06-26 | 分类于 go | 阅读次数

因为之前会C/C++和python，这里主要记录一下go和这些语言的相关性和差异，方便快速记住。

1 一些语法规则

1.1 注释

和C/C++一样，单行注释//，块注释/**/

1.2 代码分割

一行写一条语句，可以不用分号结尾。一行写多条语句，需要用分号分割。

1.3 文件后缀

文件后缀为.go

2 包

go程序是由包组成的，所以需要声明当前文件属于哪个包，引入了其它什么包。

2.1 声明包

文件非注释的第一行声明当前文件属于什么包。
package package_name

2.2 引入包

2.3 package main

每个Go应用程序必须包含一个main包,main包里的main函数是程序的入口（如果有 init 函数则会先执行该函数）

2.4 导出内容

在一个包中，首字母大写的变量和常量才能被导出。
在导入包之后，你只能访问包所导出的名字，任何未导出的名字是不能被包外的代码访问的。

3 函数

3.1 声明函数

①用func声明函数，之后依次为函数名，参数列表，返回值和函数体。

②正常是值传递，可以指针实现引用传递。
③可以有多个返回值（和python一样）。
而且，go的函数返回值支持命名，可以理解为C/C++中的出参

④函数可以赋值给变量，即可以把函数当成一种数据类型
⑤支持闭包，即匿名函数，好处在于可以直接使用函数内的变量，不必申明。

用func标识函数，而不是编译器根据语法自动识别，返回值放在参数列表后面，这些语法看起来有些不习惯，不知道是不是出于编译器整体语法分析效率的考虑。

apache支持SSL

发表于 2017-06-22 | 分类于 server | 阅读次数

1.确认apache的ssl模块是否安装

检查apache的模块路径下是否有mod_ssl.so文件。
如/usr/lib/apache2/modules/mod_ssl.so。

正常apt-get安装的版本会有此模块，
源代码编译版本需要在编译时添加参数–enable-ssl=shared –with-ssl=/usr/local/openssl

2.启用ssl模块

进入apache的配置目录，默认安装在/etc/apache2

mods-available里面是所有可用的模块，每个模块有后缀名为load和conf的两个文件。
mods-enabled里为当前启用的模块，其实里面就是建立了指向mods-available对应模块文件的软链接。

如果mods-enabled中有ssl.conf和ssl.load，则已经启用了ssl模块。
如果没有则执行a2enmod ssl启用ssl模块。

3.启用ssl site

apache可以同时支持多个site，sites-available和sites-enabled的组织方式同mods一样。
如果sites-enabled里有default-ssl文件，则支持ssl的site已经启用。
如果没有，则执行a2ensite default-ssl启用默认的ssl site配置文件。

关闭某site的命令为a2dissite site名称。

4.apache2端口文件说明

ports.conf为apache的端口相关文件。
NameVirtualHost对应sites-available中的site。
Listen为该站点监听哪个端口。

5.生成自己的密钥和证书文件

①生成私钥文件（秘钥长度为1024bits）
oopenssl genrsa 1024 > server.key
②生成请求证书文件
openssl req -new -key server.key > server.csr
请求证书文件是提供给CA机构进行认证的文件，如果不需要CA认证，用③中命令进行自认证即可。
③openssl req -x509 -days 365 -key server.key -in server.csr > server.crt
-x509 进行自认证
-days 证书有效期限

6.使用自己生成的证书文件

编辑使用ssl的site的配置文件，我这里用的是默认的/etc/apache2/sites-enabled/default-ssl
①启用ssl engine（ssl site中默认是启用的）
SSLEngine on
②使用生5中生成的文件
SSLCertificateFile /etc/ssl/certs/server.crt
SSLCertificateKeyFile /etc/ssl/private/server.key

7.重新启动apache

service apache2 restart

搭建git服务器

发表于 2017-06-18 | 分类于 git | 阅读次数

服务端：

1.安装git

  • apt get install git

2.创建git账户

  • adduser git
  • chsh -s $(command -v git-shell)
    这一步的目的是替换掉git用户的shell，使其他人不能用ssh远程登录服务器。
    所有的操作都是root在服务器本地完成后，将所有者改为git，
    从而限制了远程用户只能用git账户来获取和推送代码，而不能执行其他操作。

3.开启ssh RSA认证

  • 打开/etc/ssh/sshd_config， 将如下三项的注释打开

其中第三项的%h表示使用用户的home目录，所以使用git时的认证key文件为/home/git/.ssh/authorized_keys

4.创建git ssh目录

  • mkdir /home/git/.ssh
  • touch /home/git/.ssh/authorized_keys #填写可rsa认证远程登录的用户的公钥的文件
  • chown -R git:git /home/git/.ssh #将所有者改为git，使git可以读取其中的内容

5.创建git仓库

  • 这里项目名为GitTest，

6.加入允许rsa认证获取代码的用户公钥

  • cat id_rsa.pub >> /home/git/.ssh//authorized_keys #将用户提供的公钥加入认证文件的最后
  • 在git目录下创建一个pubkeys目录，再创建一个用户唯一标识名的目录，将他的id_rsa.pub文件放入，
    方便之后查找和统计是否给该用户开通过权限。这个不是必须的操作，只是为了方便之后的管理。
    虽然私钥的最后也有生成机器上的用户名和主机名，但是这个可能重复。

客户端：

  • ssh-keygen -t rsa #生成rsa公私钥，-t 加密类型选项

    生成的密钥对保存在家目录下的.ssh中，其中id_rsa.pub为公钥，id_rsa为私钥。

  • 服务端添加完权限后，即可clone代码了

重装虚拟机后遇到的问题

发表于 2017-06-15 | 分类于 other | 阅读次数

1.64bit的系统，VirtualBox无法显示64bit的安装环境.

打开BIOS的cpu虚拟化支持选项。

2.VirtualBox安装增强功能(可以支持分辨率调节、共享文件夹、共享剪贴板等功能）

①VirtualBox:[设备]->[安装增强功能]
②mkdir /mnt/cdrom
③mount /dev/cdrom /mount/cdrom
④cp /mount/cdrom/VBoxLinuxAdditions.run /tmp
⑤cd /tmp; ./VBoxLinuxAdditions.run
⑥重启系统：shutdown -r now

3.Ctrl + Alt + t 打开一个 Terminal

①桌面右上角用户名->[System Settings]->[Keyboard]
②选择[Shortcuts] tab，左边列表选择最下面的[Custom Shortcuts]。
③右边内容窗口点【+】,弹窗中Name填写Terminal，Command填写gnome-terminal，点Apply确定。
④右边内容窗口上面出现Terminal这一项，点击右边“disable”字样（一定要点右边，点其他位置效果不同）选择该项，同时按下Ctrl + Alt + t。

4.设置桌面背景和字体颜色

①打开一个终端，选择[Edit]->[Profile Preferences]。
②选择[Colors]tab，取消勾选的[Use colors from system theme],Background color选黑色，Text color自定义颜色#A6DEBC。

5.网络连接不可用

①设置正确的网络参数（IP、netmask、gateway）
②设置网卡为桥接模式。

6.apt-get install vim不成功

①设置正确的源（/etc/apt/sources.list)
②vim依赖vim-common，但是本机已经安装的vim-common版本不对。所以，卸载vim-common后再装vim就好了。

7.安装ssh进行远程连接

apt-get install ssh

8.没有中文输入法

apt-get install fcitx-pinyin
重登登录账户，按Ctrl+Space切换输入法

9.中文显示为方块形的乱码

①aptitude install locales #如果没安装locales，先安装一下。
②dpkg-reconfigure locales #重新配置locales

选择需要的locale：(按空格选择，选择完了按ENTER确认)
en_US.UTF8
zh_CN GB2312
zh_CN GBK GBK
zh_CN UTF-8 UTF-8

查看一下/etc/default/locale，如果LANG=en_US.UTF-8则不用修改

③如果还显示乱码，则需要安装字体：
apt-get install ttf-arphic-uming
apt-get install ttf-wqy-zenhei
